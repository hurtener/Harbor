# Harbor

A Go-native runtime for durable, steerable, event-driven AI agents.

> Harbor is in pre-V1 development. The design surface is fully landed (RFC, master phase plan, research briefs, contributor normatives); implementation is rolling out wave by wave. See `RFC-001-Harbor.md` for the architecture, `docs/plans/README.md` for the master phase plan (84 V1 + 14 post-V1 phases), and `docs/research/` for phase-planning research briefs.

## At a glance

Harbor is a four-layer system:

- **Harbor Runtime** — the orchestration kernel. Tasks, planner runtime, tools, memory, sessions, events, skills.
- **Harbor Protocol** — the canonical event/state contract between the Runtime and any client.
- **Harbor Console** — the observability + control-plane UI. Architecturally a Protocol client.
- **Harbor CLI** — the `harbor` binary. Phase 63 registers the seven settled subcommands (`dev`, `scaffold`, `validate`, `version`, `inspect-events`, `inspect-runs`, `inspect-topology`) with global `--quiet` / `--json` flags. `harbor version` (Phase 63), `harbor scaffold` (Phase 67), `harbor validate` (Phase 68), and `harbor dev` (Phase 64) are fully implemented; `harbor validate [path]` runs the in-process config loader against a YAML file (default `harbor.yaml`) without booting any subsystem and emits stable, file:line-precise errors (suitable as a CI pre-flight check); `harbor dev` boots the local Runtime + Protocol server on `127.0.0.1:18080` against `harbor.yaml`, mints an ephemeral ES256 dev-token printed to stderr, mounts the Phase 60 SSE+REST mux behind the Phase 61 JWT validator, and serves until SIGINT/SIGTERM. The remaining `inspect-*` subcommands stub-fail non-zero with a structured `{error,code,hint}` JSON pointing to their implementing phase. Subsequent phases bring hot-reload (Phase 65), draft saving (Phase 66), and the `inspect-*` family (Phases 69-70).

Three product properties are non-negotiable: multi-isolation across `(tenant, user, session)`, the Console-as-Protocol-client decoupling, and a swappable Planner.

## Testing your Harbor agent

The public `harbortest/` package gives end-users a five-function authoring surface for flow-level tests — `RunOnce`, `AssertSequence`, `AssertNoLeaks`, `SimulateFailure`, `RecordedEvents`. Import as `github.com/hurtener/Harbor/harbortest`; the package's godoc documents the surface, and `harbortest/agent_test.go` is the worked example.

To scaffold a fresh agent project that includes a `harbortest`-driven test, run `bin/harbor scaffold --name my-agent`. The output ships a production-shaped `harbor.yaml`, a `go.mod`, and a worked `EchoAgent` + test pair — see Phase 67 above.
