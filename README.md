# Harbor

A Go-native runtime for durable, steerable, event-driven AI agents.

> Harbor is in pre-V1 development. The design surface is fully landed (RFC, master phase plan, research briefs, contributor normatives); implementation is rolling out wave by wave. See `RFC-001-Harbor.md` for the architecture, `docs/plans/README.md` for the master phase plan (84 V1 + 14 post-V1 phases), and `docs/research/` for phase-planning research briefs.

## At a glance

Harbor is a four-layer system:

- **Harbor Runtime** â€” the orchestration kernel. Tasks, planner runtime, tools, memory, sessions, events, skills.
- **Harbor Protocol** â€” the canonical event/state contract between the Runtime and any client.
- **Harbor Console** â€” the observability + control-plane UI. Architecturally a Protocol client.
- **Harbor CLI** â€” the `harbor` binary. Phase 63 registers the seven settled subcommands (`dev`, `scaffold`, `validate`, `version`, `inspect-events`, `inspect-runs`, `inspect-topology`) with global `--quiet` / `--json` flags. `harbor version` (Phase 63), `harbor scaffold` (Phase 67), `harbor validate` (Phase 68), `harbor dev` (Phase 64), and `harbor inspect-events` / `harbor inspect-runs` (Phase 69) are fully implemented; `harbor validate [path]` runs the in-process config loader against a YAML file (default `harbor.yaml`) without booting any subsystem and emits stable, file:line-precise errors (suitable as a CI pre-flight check); `harbor dev` boots the local Runtime + Protocol server on `127.0.0.1:18080` against `harbor.yaml`, mints an ephemeral ES256 dev-token printed to stderr, mounts the Phase 60 SSE+REST mux behind the Phase 61 JWT validator, and serves until SIGINT/SIGTERM; `harbor inspect-events` tails or snapshots the Phase 60 SSE event stream (identity-scoped, with optional `--type`/`--run`/`--since` filters, human or `--json` NDJSON output) and `harbor inspect-runs` derives per-run summaries (list mode) or per-event trajectories (single-run mode) from the same SSE replay â€” both use a Bearer JWT from `HARBOR_TOKEN` or `~/.harbor/token`. The remaining `inspect-topology` subcommand stub-fails non-zero with a structured `{error,code,hint}` JSON pointing to its implementing phase. Subsequent phases bring hot-reload (Phase 65), draft saving (Phase 66), and the ASCII topology renderer (Phase 70).

Three product properties are non-negotiable: multi-isolation across `(tenant, user, session)`, the Console-as-Protocol-client decoupling, and a swappable Planner.
