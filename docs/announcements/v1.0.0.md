**Harbor v1.0.0 is out — a Go-native runtime for durable, steerable,
event-driven AI agents.**

Most agent frameworks are a loop: call the model, run a tool, repeat, return a
string. That loop is easy to start and hard to operate. It can't be paused for
an approval, it loses everything on a crash, two users share its globals, and
the only window into it is stdout.

Harbor was built to make those operational concerns the product.

## What v1.0.0 gives you

- **Durable runs.** Run state is persisted, not held in a goroutine. A run
  survives a process restart. Pause/resume serialization fails *loudly* rather
  than silently dropping context.
- **Steerable runs.** Cancel, redirect, inject a message, pause, resume — nine
  control operations through one primitive. Human-in-the-loop approval,
  tool-side OAuth, and operator pause are the *same* mechanism, not three.
- **Event-driven by construction.** Every meaningful thing a run does is a
  typed event on a bus. Observability isn't bolted on — it's the wiring.
- **Multi-isolation from line one.** Every layer carries a
  `(tenant, user, session)` identity. One user holds many concurrent sessions
  with zero cross-talk. There is no single-tenant mode to outgrow.
- **A swappable planner.** The runtime owns mechanism; the planner owns
  reasoning policy, behind one interface. The reference ReAct planner ships in
  the box, with a Deterministic planner beside it to prove the seam.

It ships as one importable Go module and one static binary. No CGo, no broker
to stand up, no daemon to operate. `harbor dev` boots the whole runtime on a
laptop in under a second; `harbor scaffold` writes a runnable agent and test.

## Built to be operated

Harbor's runtime, the versioned Harbor Protocol, the Console, and the CLI are
four layers with hard boundaries — the Console is *only* a Protocol client, so
the same surface powers a remote attach or a third-party dashboard. Persistence
ships as three conformance-equal drivers (in-memory, SQLite, Postgres). Tools
register the same way whether they're an in-process Go function, an HTTP
endpoint, an MCP server, or an A2A agent.

Every change to Harbor is gated by conformance harnesses that hammer
cross-tenant isolation, goroutine-leak freedom, and fault recovery under the
race detector. v1.0.0 is the line where that surface is complete and stable.

## Get started

```bash
go install github.com/hurtener/Harbor/cmd/harbor@v1.0.0
harbor scaffold --name my-agent && cd my-agent && harbor dev
```

The [README](../../README.md) has the full tour; [`docs/recipes/`](../recipes/)
has copy-paste how-to guides; [`RFC-001-Harbor.md`](../../RFC-001-Harbor.md) is
the design rationale, decision by decision.

## What's next

v1.0.0 is a foundation, not a finish line. Post-V1 work — deeper ReAct
prompting, additional planner concretes (Plan-Execute, Graph, Supervisor), a
durable distributed bus, and governance extensions — builds on exactly these
primitives. Issues and discussion welcome.
